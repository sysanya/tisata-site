# TM TISATA — сайт-магазин обуви

Стильная витрина обуви (женская + мужская) в эстетике нишевых брендов.
Заказ оформляется в **Telegram** одним готовым сообщением. Онлайн-оплату можно добавить позже.

Сайт статический — работает без сервера, размещается бесплатно.

---

## 1. Как посмотреть сайт у себя

Просто откройте файл `index.html` двойным щелчком в браузере.

Для корректной работы корзины лучше запустить локальный сервер (одна команда в терминале, находясь в папке `tisata-site`):

```bash
python3 -m http.server 8080
```

Потом откройте в браузере: http://localhost:8080

---

## 2. Как настроить под себя

Откройте файл `assets/js/config.js` и поменяйте значения:

- `brand` — название бренда
- `telegram` — ваш username в Telegram (без `@`) — **туда придут заказы**
- `instagram`, `email`, `city` — контакты
- тексты на главном экране (`heroLine1`, `heroLine2`, `heroSubtitle`)

> Важно: впишите свой настоящий Telegram-username в `telegram`, иначе кнопка «Оформить» отправит заказ в несуществующий аккаунт.

## 3. Управление товарами через Google-таблицу (рекомендуется)

Вы ведёте товары в обычной таблице. Поменяли цену / убрали размер / поставили «нет» в
колонке «Наличие» — сайт обновляется сам при загрузке страницы. **Без правки кода и без
перезаливки сайта.**

### Шаг 1. Создать таблицу
1. Зайдите на https://sheets.google.com и создайте новую таблицу.
2. Меню **Файл → Импорт → Загрузка** и выберите файл `товары-шаблон.csv` из этой папки
   (в нём уже лежат ваши 2 модели). При импорте выберите «Заменить текущий лист».

### Шаг 2. Опубликовать таблицу
1. Меню **Файл → Поделиться → Опубликовать в интернете**.
2. Выберите: лист — «Товары» (или весь документ), формат — **CSV**.
3. Нажмите «Опубликовать» и скопируйте полученную ссылку
   (вида `https://docs.google.com/spreadsheets/d/e/XXXX/pub?output=csv`).

### Шаг 3. Подключить к сайту
Откройте `assets/js/config.js` и вставьте ссылку:
```js
sheetCsvUrl: "https://docs.google.com/spreadsheets/d/e/XXXX/pub?output=csv",
```
Готово. Теперь сайт берёт товары из таблицы.

### Колонки таблицы
| Колонка | Что писать |
|---|---|
| **id** | уникальный номер (1, 2, 3 …) |
| **Название** | название модели |
| **Категория** | `женское` / `мужское` / `унисекс` |
| **Цена** | число, например `25500` |
| **Старая цена** | для зачёркивания; `0` если скидки нет |
| **Бейдж** | `NEW`, `ХИТ`, `-20%` или пусто |
| **Размеры** | через `;` или `,`: `36; 37; 38`. **Указывайте только те размеры, что есть в наличии** |
| **Цвета** | через `;`: `Чёрный; Белый` |
| **Фото** | имена файлов из папки `images/` через `;`, либо ссылки на фото |
| **Описание** | текст |
| **Наличие** | `да` или `нет` (нет → бейдж SOLD OUT, кнопка заказа отключается) |

> Как обновлять склад: уберите распроданный размер из колонки «Размеры», либо поставьте
> «нет» в «Наличие», чтобы скрыть продажу всей модели.

### Фото для новых товаров
Текст (цены, размеры, наличие) меняется в таблице мгновенно. А вот **фото нового товара**
нужно один раз положить в папку `images/` и заново опубликовать сайт (либо указать в колонке
«Фото» прямую ссылку на изображение в интернете — тогда перезаливка не нужна).

### Запасной вариант без таблицы
Если `sheetCsvUrl` пустой — сайт берёт товары из файла `assets/js/products.js`
(там тоже можно править вручную). Это «резервный» список на случай, если таблица недоступна.

## 4. Публикация на tisata.ru

**Основной путь:** GitHub + Netlify + домен. Подробно → **[DEPLOY.md](./DEPLOY.md)**

Кратко:
1. `git push` в GitHub
2. Netlify подключён к репо — деплой автоматически
3. Домен **tisata.ru** → DNS на Netlify

**Быстрый тест без домена:** https://app.netlify.com/drop — перетащить папку.

**Обновление:** изменил файлы → `git add .` → `git commit` → `git push` (~1 мин до продакшена).
